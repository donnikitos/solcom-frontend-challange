# Frontend Consultant Interview Challenge

## Overview

Welcome to the frontend code challenge! In this task, you will build a frontend application that interacts with a provided SQLite database via given API endpoints. You will have **2 hours** to complete the challenge and submit your solution as a zip file back to the person who send you this case.

## Setup

1. `npm install`
1. `cd backend && npm install`
1. `cd frontend && npm install`
1. `npm run start`

## Provided Resources

-   **Database**: A preconfigured SQLite database is provided.
-   **API**: The API endpoints are ready to use for all necessary operations.

## Challenge Tasks

Your task is to build a user-friendly and fully type-safe frontend that meets the following requirements:

1. **Display Inventory**
   Create an interface that displays the inventory in a clear and user-friendly manner.

2. **Update Stock Quantity**
   Implement functionality to update the stock quantity of a specific item.

3. **Update Stock Price by Category**
   Add the ability to update the stock price across all items within a specific category.

4. **Search Functionality**
   Allow users to search for an item by its name and category.

5. **Filtering Functionality**
   Implement filters for items based on price and stock quantity.

6. **Chart Display**
   Build a chart that displays the updates and filtered results dynamically. Be creative on which information you want to provide in which way.

7. **Loading Indicators**
   Let the user know when the API is loading to improve the user experience.

8. **Responsiveness**
   Ensure that the application is fully responsive across different devices.

9. **Styling**
   Apply a consistent styling across your web application using a modern toolstack.

## Some notes

1. Feel free to deviate from the proposed framework and toolstack as long as you can sensibly justify your choices.
2. In case you have time issues, prioritize features which you think provide the most user experience.
3. `generateData.ts` is just provided for documentation purposes and if you want to recreate the test database
